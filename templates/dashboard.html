PRO TIPS
🔁 Re-exports are safe
Push the same data twice and nothing breaks. Standings, rosters, and stats upsert by Madden's own row IDs — duplicates are deduped automatically.
🏟️ One server per league
Each Discord server maps to one league via /setup league league_id:<id>. Running multiple franchises? Create one league here per server.
📅 Pre-/Reg-/Post-season
The Companion App tags exports with the season phase. Filter the API or bot output by season_type=pre|reg|post when you need a specific window.
⚡ Auto-export every sim
Turn on Auto-Export in the Companion App. The bot can auto-post matchup previews and weekly recaps — no manual ingest needed.
COMMON QUESTIONS
Do I have to use the Discord bot?
No — the web dashboard works on its own. You can browse standings, rosters, schedule, and leaders from any league page. The bot adds slash commands and AI-generated weekly news on top.
What if I run multiple Madden leagues?
Create one league here per franchise. Each gets its own ID and Madden League ID — exports from the Companion App route by Madden League ID, so leagues don't mix even if they share a Discord account.
Where do I get a Discord bot token?
Visit discord.com/developers/applications, create a new app, go to the Bot tab, click Reset Token, and copy. Drop it in the bot's .env as DISCORD_BOT_TOKEN. Detailed steps live in the bot repo's README.
How is my API key used?
The API key authenticates direct calls to /api/{league_id}/... for read/write access. The Madden Companion App does not need it — it routes by Madden League ID. Only share the key with services you control.
My export went through but standings are blank — why?
The Companion App exports each category separately. Make sure you've exported Standings at least once (Settings → Export → Standings). Schedules, rosters, and stats are independent endpoints.
Can I delete a league?
Not from the UI yet — open an issue if you need this. For now, leagues are kept so you can keep historical data when re-importing later.
